vscode配置Cplusplus

发表于2020-12-29|更新于2020-12-29|VScode
|字数总计:1.6k|阅读时长:5分钟|阅读量:|评论数:

预备条件(Prerequisites)

  1. 安装Visual Studio Code
  2. 安装拓展C/C++
  3. 通过SourceForge网站安装MinGW,点击链接下载Windows Mingw-w64安装程序。
    • 运行安装程序
    • Architecture选择x86_64然后点击Next
    • 在安装文件夹(Installation Folder)页面,可以自己选择文件夹,记住该位置后面将会用到
    • 点击Next开始安装
  4. 将bin文件夹的路径添加到Windows的PATH环境变量当中

检查MinGW的安装

为了检测你的Mingw-w64工具被正确地安装并且是有效的,打开命令行输入:

g++ --version
gdb --version

如果你没有看到期望输出或者g++或gdb为不被识别的命令的话，你需要检查你的环境变量的添加是否有误。

创建Hello World

通过Windows的命令行创建名为projects的空文件夹，这里可以用来放置你所有的VScode项目.然后创建名为helloworld的子文件夹，进入该文件夹，然后用VScode打开该文件夹：

mkdir projects
cd projects
mkdir hellowprld
cd helloworld
code .

其中的code .命令会让VScode打开当前文件夹,而这个文件夹就成为了你的工作空间(workspace)。

添加源代码文件

在VScode文件资源管理器的标题栏点击New File按钮新建文件并将文件命名为helloworld.cpp，回车，进行代码编辑，如下为示例代码:

#include<iostream>
using namespace std;
int main()
{
    cout<<"Hello World.<<endl;
    return 0;
}

可以使用快捷键Ctrl+S保存代码，也可以将文件菜单中的Auto Save勾选完成自动保存。

Build helloworld.cpp

接下来，你会创建一个tasks.json文件用来告诉VS Code怎样编译该程序：这个task会援引g++编译器创建一个基于源代码的可执行文件。

在主菜单中选择终端>配置默认生成任务，在下拉菜单中选择g++.exe build active file，同时这会在.vscode文件夹中生成文件tasks.json，在编辑器中将其打开，你的新tasks.json文件看起来与下面的类似:

{
"version": "2.0.0",
"tasks": [
	{
		"type": "cppbuild",
		"label": "C/C++: g++.exe build active file",
		"command": "E:\\MinGW\\mingw64\\bin\\g++.exe",
		"args": [
			"-g",
			"${file}",
			"-o",
			"${fileDirname}\\${fileBasenameNoExtension}.exe"
		],
		"options": {
			"cwd": "E:\\MinGW\\mingw64\\bin"
		},
		"problemMatcher": [
			"$gcc"
		],
		"group": {
			"kind": "build",
			"isDefault": true
		},
		"detail": "compiler: E:\\MinGW\\mingw64\\bin\\g++.exe"
	}
]
}

其中的command确定了要运行的程序，在这里是g++。args确定了将被传给g的命令行参数，这些参数必须按照编译器期望的顺序给定。此task告诉g获取活动文件(${file})，将其编译然后在当前目录(${fileDirname})创建与活动文件同名但是拓展名为.exe的可执行文件(${fileBasenameNoExtension}.exe)，在我们的例子里面就是helloworld.exe。

label的值就是你将在任务列表中看到的东西，可以随意命名。

group中的"isDefault": true表示可以通过快捷键Ctrl+Shift+B进行运行，这只是为了方便；如果将之设置为false，仍然可以通过点击终端>运行生成任务进行运行。

运行(Running the build)

  1. 返回到helloworld.cpp

  2. 为了运行定义在tasks.json中的已建立任务,按Ctrl+Shift+B或者在终端主菜单中选择运行任务

  3. task开始后，会看到源代码编辑器下方出现集成终端面板，当task完成后，终端显示从编译器传来的输出，如果编译成功，输出看起来类似下面:

    Executing task: C/C++: g++.exe build active file <
    Starting build…
    Build finished successfully.

  4. 另外创建一个新的终端，你会看到该终端以helloworld文件夹为工作目录，运行dir你会看到可执行文件helloworld.exe。

  5. 在终端中键入helloworld.exe(如果使用的是PowerShell终端请键入.\helloworld.exe)来运行helloworld。

修改tasks.json

你可以修改你的tasks.json文件,比如你可以将${file}修改为${workspaceFolder}\\*.cpp来生成当前文件夹中的所有.cpp文件。

调试helloworld.cpp

接下来，可以创建launch.json文件来配置VS Code使得当你按F5来调试程序时VS Code会启动GDB调试器。

  1. 在主菜单选择运行>添加配置，然后选择C++(GDB/LLDB)。
  2. 你会看到下拉菜单，选择g++.exe build and debug active file。
    VS Code会创建launch.json文件，在编辑器中将其打开。

program选项确定了你要调试的程序。

默认情况下，C++拓展不会为你的源代码添加任何断点并且stopAtEntry的值被设置为false

将stopAtEntry的值设置为true会导致当你开始调试时调试器会在main方法前暂停一次

注意：preLaunchTask的设置被用来确定开始调试之前要被执行的task，因此要确保其和tasks.json文件中label的设置相一致。

C/C++配置

如果你想要对C/C拓展进行更多的控制，你可以创建一个c_cpp_properties.json文件，这个文件允许你改变一些设置比如编译器的路径，包含路径(include paths)，C标准和其它的一些设置。

你可以通过在命令面板中键入C/C++:Edit Configurations(UI)来进入C/C++配置用户界面，当你在这个进行一些设置，VS Code会将它们写进.vscode文件夹中一个名为c_cpp_properties.json的文件中。

将配置名改为GCC(可以不改)，将编译器路径设置为你的电脑上g++.exe所在路径，我的路径是

E:/MinGW/mingw64/bin/g++.exe

将智能感知模式(IntelliSense mode)设置为gcc-x64以匹配编译器。

当你直接打开文件c_cpp_properties.json，会看到它类似下面：

{
"configurations": [
    {
        "name": "GCC",
        "includePath": [
            "${workspaceFolder}/**"
        ],
        "defines": [
            "_DEBUG",
            "UNICODE",
            "_UNICODE"
        ],
        "compilerPath": "E:/MinGW/mingw64/bin/g++.exe",
        "cStandard": "gnu11",
        "cppStandard": "c++17",
        "intelliSenseMode": "gcc-x64"
    }
],
"version": 4
}

你仅仅需要做的是，当你的程序需要的头文件不在你的工作空间和标准库路径中时，你将头文件的路径添加到Include path中。
